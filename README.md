## ⚡ なぜ Next.js ではなく Vite を採用したのか？

今回のプロジェクト要件（診断LP・シングルページアプリケーション）において、Next.jsのような多機能なフレームワークよりも、**Vite**の方が最適であると判断しました。

### 1. SPA（シングルページアプリケーション）に特化
このアプリは「ページ遷移」を伴う複雑なサイト構造ではなく、ユーザーがその場で診断を行い結果即座に表示する**SPA**形式です。サーバーサイドレンダリング（SSR）や静的サイト生成（SSG）などのNext.jsの強力な機能は、この構成ではオーバースペック（過剰機能）となり、逆に複雑さを生む原因となります。

### 2. 軽量かつ高速なパフォーマンス
- **開発スピード**: Viteは開発サーバーの起動が一瞬で、コード修正時の反映（HMR）も爆速です。
- **バンドルサイズ**: Next.jsはサーバー機能を含むためバンドルサイズが大きくなりがちですが、ViteでビルドされたReactアプリは静的なHTML/JS/CSSのみになるため非常に軽量です。

### 3. デプロイの柔軟性
Viteの成果物は純粋な静的ファイル（HTML/JS）です。そのため、VercelやNetlifyはもちろん、GitHub PagesやAWS S3など、**あらゆる静的ホスティング環境**に簡単に配置できます。Node.jsサーバーを常時稼働させる必要がないため、運用コストも最低限に抑えられます。
